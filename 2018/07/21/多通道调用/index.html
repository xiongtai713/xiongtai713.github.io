fabric多通道应用

2018-07-21 | 阅读：次

多通道调用

1. 首先创建应用通道交易配置文件
2. 创建组织锚节点更新文件
3. 启动网络并进入CLI节点
4. 创建指定的新的应用通道
5. 将节点加入至通道中

###临时变量 同到名称

$ export CHANNEL_NAME=mychannel2

###创建新通道配置文件

$ sudo ../bin/configtxgen -profile TwoOrgsChannel -outputCreateChannelTx ./channel-artifacts/channel.tx -channelID $CHANNEL_NAME
  1. 工具名称 configtxgen
  2. 通道模板 -profile TwoOrgsChannel 路径在first-network/configtx.yaml中 模板内是哪个组织可以加入通道
  3. outputCreateChannelTx(创建新的配置文件) ./channel-artifacts(文件储存在)/channel.tx(储存的文件名称)
  4. 通道ID -channelID $CHANNEL_NAME (这个名称会保存在channel.tx这个文件里) ###创建通道内的锚节点
$ sudo ../bin/configtxgen -profile TwoOrgsChannel -outputAnchorPeersUpdate ./channel-artifacts/Org1MSPanchors.tx -channelID $CHANNEL_NAME -asOrg Org1MSP

$ sudo ../bin/configtxgen -profile TwoOrgsChannel -outputAnchorPeersUpdate ./channel-artifacts/Org2MSPanchors.tx -channelID $CHANNEL_NAME -asOrg Org2MSP
  1. 工具名称 configtxgen
  2. 通道模板 -profile TwoOrgsChannel
  3. outputAnchorPeersUpdate ./channel-artifacts(保存路径)/Org2MSPanchors.tx(储存文件名称)
  4. -channelID $CHANNEL_NAME(这个通道id必须跟通道配置的id一样,否则无法加入通道)
  5. -asOrg Org2MSP 这个节点是哪个组织的(Org2MSP 是org2组织的) ?创建锚节点时 Org2MSP 是关键字吗(在congfig里有组织名称) ###启动网络
sudo docker-compose -f docker-compose-cli.yaml up -d

进入cli操作节点

docker exec -it cli bash
进入cli之后 还要设定临时变量
$ export CHANNEL_NAME=mychannel2

创建通道

peer channel create -o orderer.example.com:7050 -c $CHANNEL_NAME -f ./channel-artifacts/channel.tx --tls --cafile /opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem
  1. create -o orderer.example.com:7050 (指定orderer节点)
  2. -c $CHANNEL_NAME (通道id)
  3. -f ./channel-artifacts/channel.tx (通道配置文件,声明那些组织可以加入本通道)
  4. –tls(开启tls验证验证ca证书) –cafile /opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem (ca证书地址 证书被包含在组织结构里面了,映射到crypto文件夹里,直接使用orderer节点下的ca证书)

创建完成后产生.block通道文件 列出当前节点已加入的通道 peer channel list 在CLI容器中查看当前操作的是哪个节点 echo $CORE_PEER_ADDRESS

###节点加入新创建的通道

peer channel join -b mychannel2.block -o orderer:7050

加入通道后可以进行查询 如果别的节点要加入通道,进行节点的切换 节点在通道实例化链码的时候,一定注意声明在哪个通道上实例化
