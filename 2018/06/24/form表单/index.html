form表单

HTML 表单用于搜集不同类型的用户输入。

form: 使用的是<form> 元素定义 HTML 表单，不同类型的表单元素如： input 元素、复选框、单选按钮、提交按钮等等。

input: 元素有很多形态，下面是不同的 type 属性值。

类型 描述
text 定义常规文本输入
radio 定义单选按钮输入（选择多个选择之一）
submit 定义提交按钮（提交表单）
password 定义密码字段

form属性：

属性 描述
accept-charset 规定在被提交表单中使用的字符集（默认：页面字符集）。
action 规定向何处提交表单的地址（URL）（提交页面）。
autocomplete 规定浏览器应该自动完成表单（默认：开启）。
enctype 规定被提交数据的编码（默认：url-encoded）。
method 规定在提交表单时所用的 HTTP 方法（默认：GET）。
name 规定识别表单的名称（对于 DOM 使用：document.forms.name）。
novalidate 规定浏览器不验证表单。
target 规定 action 属性中地址的目标（默认：_self）。
value 规定输入字段的初始值。
readonly 属性规定输入字段为只读（不能修改）

**action: **

action 属性定义在提交表单时执行的动作。向服务器提交表单的通常做法是使用提交按钮。通常，表单会被提交到 web 服务器上的网页。

method:

method 属性规定在提交表单时所用的 HTTP 方法（GET 或 POST）。

GET是默认方法），如果表单提交是被动的（比如搜索引擎查询），并且没有敏感信息。当您使用 GET 时，表单数据在页面地址栏中是可见的。GET 最适合少量数据的提交。浏览器会设定容量限制。

如果表单正在更新数据，或者包含敏感信息（例如密码）。POST 的安全性更加，因为在页面地址栏中被提交的数据是不可见的。

name:

如果要正确地被提交，每个输入字段必须设置一个 name 属性。表单提交只会提交带有name属性的元素

示例代码：

    <meta charset="UTF-8">
    <base href="http://localhost:2000/">

<form action="login" method="post" target="_top" accept-charset="UTF-8"
      enctype="application/x-www-form-urlencoded" onsubmit="return checkForm()">
    用户名：<br><input type="text" id="username3" name="username" value="kong"><br>
    密码：<br><input type="password" id="password3" name="password" value="123456" ><br>
    <button type="submit">登录</button>
</form>
